using cipherVector = lbcrypto::Ciphertext<lbcrypto::DCRTPoly>

Author: Ian Quah Date: 18 Dec

{Although the name may suggest it, we only support rank 2 tensors and below For the most part we try and mimic the numpy interface as closely as possible.

1) NOTE: we always require that the LHS is a ciphertext. The RHS may be either a ciphertext or raw message E.g in x - y x + y x must be a ciphertext and y can be either a ciphertext or raw message

For now we only return an encrypted tensor. We will (maybe) support other types in the future

2) We don't distinguish between a scalar or a vector in the encrypted sense as we just have a ciphertext. We do, however, have a concept of a vector of ciphertexts which we call a ciphertext matrix

3) If we are passed in a message we by-default also generate the transpose (which is then encrypted). So, by default all pTensors which are passed in plaintexts will have their encrypted transpose for free. We also provide a function to generate the transpose for encrypted matrices but it is much slower. } Main Palisade Tensor class

  • We need to initialize a static member before doing any of the operations
